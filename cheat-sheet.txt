
~ домашняя директория
.. родительская директория

pwd показать адрес текущей директории

ls показать содержимое директории
ls -a показать содержимое директории + скрытое
ls ~ показать содержимое родительской директории
ls .. показать содержимое директории выше уровнем
	Навигация
cd сменить директорию
cd ~ перейти в домашнюю директорию
cd .. перейти на уровень выше
	Создание файлов и папок
touch 'new file.txt' создать текстовый файл
mkdir 'new dir' создать папку new dir
mkdir -p dir/new-dir/3dir создать структуру директорий
touch и mkdir можно использовать с символами (~) и (..)
	Коприрование и перемещение
cp копировать
cp что копируем куда копируем
копировать можно несколько файлов одновременно
$ cp index.html style.css script.js src/

mv (move) переместить
синтаксис аналогичен команде копировать
	Чтение и удаление
cat (concatenate and print - объединить и распечатать) чтение файлов
чтение только текстовых файлов

rm (remove - удалить) - удалить файл
rmdir - удалить директорию (только елсли она пуста)
rm -r - удалить папку и всё содержимое

&& можно объединять команды

# создаём папку second-project,
# переходим в папку second-project
# и создаём в ней два файла: index.html и style.css

	Настройки git
.gitconfig файл в котором хранятся настройки git

git init - сделать папку репозиторием

Разгитить папку, если что-то пошло не так, — rm -rf .git
Проверить состояние репозитория — git status
Подготовить файлы к сохранению — git add

	Хеш — идентификатор коммита
Git преобразует информацию о коммитах с помощью алгоритма SHA-1 и для каждого из них рассчитывает уникальный идентификатор — хеш.
Хеш — основной идентификатор коммита и позволяет узнать его автора, дату и содержимое закоммиченных файлов.
Все хеши, а также таблицу соответствий хеш → информация о коммите Git хранит в папке .git.

	Лог $git log
Можно вызвать не только полный лог, но и сокращённый — это делается командой git log --oneline.
В сокращённом логе выводятся сокращённые хеши — их можно использовать точно так же, как и полные.

	Head
Папка .git содержит много непонятных файлов — об одном из них мы рассказали в этом уроке. Подытожим:
В числе прочих файлов в папке .git есть служебный файл HEAD. Он указывает на самый свежий коммит.
Вместо хеша последнего коммита можно написать слово HEAD — Git вас поймёт.
